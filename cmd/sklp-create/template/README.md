# app

Skalpai-style application generated from the `stack` template. Ships
with the `sklp` dev/CI/publish workflow, OTEL observability via skalpai
sdk-go, a Go DDD backend (`apps/core`) with Postgres + NATS JetStream
event consumers, Better Auth (`@lalternative/auth`), a typed React Query
API client generated by orval into the shared front lib (`lib/front`),
and a React + TanStack web app (`apps/web`). Drop the parts you don't
need (e.g. the NATS wiring) after scaffolding.

## Quick start

```bash
cp .env.example .env                # fill SKALPAI_PROJECT_ID + SKALPAI_API_KEY
sklp dev stack                      # boots core (4100) + web (5273)
```

## Layout

```
.sklp/                 space.yaml + stack/dev.yaml + pipelines/{ci,build,generate,publish,secops}.yaml
apps/
  core/                Go 1.25 + Echo v4 + Postgres + NATS (DDD per bounded context)
  web/                 React 19 + TanStack Start (SSR via Nitro)
lib/front/             Shared front lib: typed React Query client (@app/front)
Dockerfile*            One per shipped image (core distroless, web = Nitro SSR server)
```

## API client

The core exposes an OpenAPI spec (swaggo annotations on the Echo handlers).
orval generates a typed **React Query** client from it into the shared front
lib `lib/front/src/api` (package `@app/front`), routed through a hand-written
`fetch` mutator (`lib/front/src/orval-fetcher.ts`) that sends the session
cookie same-origin via the web's `/api/v1/*` reverse proxy. The client is consumed
as source (no build step). Never hand-edit it — annotate the handlers, then
regenerate:

```bash
sklp run generate    # swag → apps/core/docs, orval → lib/front/src/api
```

Consume per-tag hooks from the package subpath, e.g.
`import { useCreateExample } from "@app/front/api/examples/examples"`.
`apps/core/docs` and `lib/front/src/api` are checked in.

## Workflow

Source of truth: `.sklp/space.yaml` (runner recipe), `.sklp/stack/dev.yaml`
(dev topology) + `.sklp/pipelines/*.yaml` (CI/build/publish).

| Command | What it runs |
|---|---|
| `sklp dev stack` | Local supervisor — core + web |
| `sklp run ci` | Lint + test impacted services |
| `sklp run generate` | Regenerate OpenAPI spec + typed API client (swag → orval) |
| `sklp run build` | Build `bin/core` + web bundle |
| `sklp run secops` | Security scan: gitleaks + semgrep + govulncheck + trivy |
| `sklp run publish` | Build, scan (Trivy), push impacted images (main only) |

**No GitHub Actions** drive `sklp` pipelines. CI runs via `sklp flow end`
before a PR (and again on the PR). Remote dispatch is the skalpai UI.

## Observability

`apps/core/observability/` wraps `github.com/lalternative/packages/skalpai/sdk-go`
and configures OTEL traces + metrics + logs at boot. Traffic flows to the URL
in `SKALPAI_ENDPOINT` (using `SKALPAI_API_KEY`). No-op when either env var is
empty.

## Image policy

`rg.fr-par.scw.cloud/skaplai/<service>:YYYYMMDD-HHMM-<short-sha>` — immutable.
No `latest`. Trivy gates HIGH/CRITICAL fixable findings.
